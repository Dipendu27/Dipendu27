## 📌 Featured Project — 30-Day DevOps Cloud Testing Journey

<p align="center">
  <a href="https://github.com/Dipendu27/devops-cloud-testin">
    <img src="https://img.shields.io/badge/Repository-devops--cloud--testin-0E75B6?style=for-the-badge&logo=github&logoColor=white" />
  </a>
  <img src="https://img.shields.io/badge/Stack-Bash%20%7C%20Docker%20%7C%20GitHub%20Actions%20%7C%20AWS-success?style=for-the-badge" />
</p>

A 30-day hands-on transition from Manual QA to Cloud Automation Engineer.  
Every script, container, pipeline, and deployment was built and debugged in real-time.

### What I Built

| Project | Description | Tech |
|---|---|---|
| **Ubuy E-Commerce Monitor** | Monitors live endpoints, handles 5 HTTP status types, logs alerts with timestamps | Bash, curl, cron |
| **Newman API Test Suite** | 3 endpoints, 6 assertions, HTML report generation | Newman, Postman, Node.js |
| **Docker Test Environment** | Alpine-based container — identical execution on Mac, Ubuntu, and CI | Docker |
| **GitHub Actions Pipeline** | Full CI/CD: build → test → report → artifact upload on every push | GitHub Actions |
| **AWS Cloud Orchestration** | EC2-hosted runner with Docker execution and S3 log archiving via IAM | AWS EC2, S3, IAM |
| **Cloud Cron Scheduler** | `*/30` and `@reboot` jobs on EC2 for 24/7 zero-touch test execution | Linux cron |
| **Dynamic JSON Validator** | Parses live inventory arrays with `jq` loops, raises out-of-stock alerts | Bash, jq |
| **LLM Benchmark Script** | Measures Ollama model response time, token count, tokens/sec | Python, Ollama |

### Pipeline Architecture

```
git push
    ↓
GitHub Actions triggered
    ↓
Docker image built (Alpine Linux)
    ↓
Ubuy endpoint monitor runs → logs generated
    ↓
Newman API test suite runs → HTML report generated
    ↓
Artifacts uploaded to GitHub
    ↓
EC2 cron job (every 30 min) → Docker → S3 archive
```
